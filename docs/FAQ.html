    • 每个 action 都调用 “所有的 reducer” 会不会很慢？
    • 在 reducer 中必须对 state 进行深拷贝吗？拷贝 state 不会很慢吗？
    • 怎样减少 store 更新事件的数量？
    • 仅有 “一个 state 树” 会引发内存问题吗？分发多个 action 会占用内存空间吗？

    • 为何组件没有被重新渲染、或者 mapStateToProps 没有运行？
    • 为何组件频繁的重新渲染？

    • 为何不在被连接的组件中使用 this.props.dispatch ？
